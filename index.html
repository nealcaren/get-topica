Why topica

Everything reviewers ask for, in one install.

General toolkits hand you topics. topica is built around the question social scientists ask next — how prevalence and content relate to covariates — with the validation and reporting to defend it.

The shelf

39 models, one namespace

LDA, STM, CTM, keyATM, SAGE, HDP, dynamic, neural, and embedding-based — models usually split across MALLET and R, all behind import topica.

No heavy runtime

A single wheel

A Rust core with a NumPy-native API. It needs only NumPy and pandas — no Java, no PyTorch, no build step.

Trust the fit

Validated & reproducible

Every model is checked against its reference implementation. Variational fits are identical to the bit; samplers reproduce from a fixed seed and thread count.

Covariates

Effects, not just topics

Relate topic prevalence and content to metadata with structural models and well-calibrated hypothesis tests — estimate_effect out of the box.

Honest uncertainty

An error bar on everything

Standard errors and confidence intervals next to coefficients, coherence, correlations, and ideal-point positions — uncertainty is a first-class citizen.

Your data

DataFrame in, results out

from_dataframe keeps metadata aligned to the documents that survive pruning, so the same corpus feeds the model, the effects, and the report.

Show, don't tell

An error bar on every estimate.

Topics are estimated, not observed — so topica carries that uncertainty into every number downstream. Below: a real fit relating an experimental treatment to each topic's prevalence, with 95% intervals from the method of composition (Rubin-pooled over posterior draws, the procedure R stm uses).

  • Interval clears zero — a real shift. The treatment raises jobs / security and lowers country.
  • Interval spans zero — no detectable effect. Three of the five topics simply don't move, and the bars say so.

This whole figure is one call:
estimate_effect(draws, X, ci=0.95).

Quick start

Runs exactly as written, right after install.

On a bundled example dataset — no download, no API key, no configuration.

01

Load & build a corpus

Point topica at a DataFrame column; stopwords and pruning are one argument each.

02

Fit a model

Sensible defaults, no tuning required. Swap LDA for any of the 39.

03

Read & relate

Top words per topic, then relate prevalence to a covariate with a calibrated test.

quickstart.py 
import topica

df     = topica.datasets.load_gadarian()        # bundled, offline
corpus = topica.from_dataframe(
    df, text_col="open.ended.response",
    stopwords=topica.ENGLISH_STOPWORDS,
)

model = topica.LDA(num_topics=5, seed=42)
model.fit(corpus)
print(topica.summary(model))                  # top words

# relate topic prevalence to a covariate
stm = topica.STM(num_topics=5, seed=42)
stm.fit(corpus, prevalence, prevalence_names=["treatment"])
effect = topica.estimate_effect(draws, prevalence,
                                feature_names=["treatment"])
Speed

Fast enough to iterate.

A Rust core running collapsed-Gibbs sampling — no JVM to warm up, no GPU to schedule. Change a setting and refit before your train of thought moves on.

Manifesto

Built for the paper, not the demo.

cat manifesto.txt

A topic model is only as good as what you can defend about it.

So topica ships the boring parts that make a finding hold up: every model checked against its reference implementation, fits that reproduce from a seed, an error bar next to every estimate, and a straight line from your DataFrame to the effect you report.

No JVM to install. No cluster to rent. No black box to apologize for. Open source, Apache-2.0, and made for the questions computational social scientists actually ask.
